🎯 Objetivo da atividade

O que você vai construir?

Uma clínica médica precisa de um sistema em Python para gerenciar seus pacientes. Existem dois tipos: pacientes particulares (pagam diretamente pelo atendimento) e pacientes por convênio (atendidos via plano de saúde). Cada tipo tem características e comportamentos distintos, mas compartilham uma base comum.

Você vai modelar essa realidade usando os pilares da Programação Orientada a Objetos: uma superclasse Paciente com atributos e métodos comuns, e duas subclasses que herdam e estendem esse comportamento com suas particularidades.

💡 Leia todo o enunciado antes de começar a codificar — a estrutura foi pensada para se encaixar.

Classe Base — Paciente

características comuns a qualquer paciente da clínica
Superclasse

Representa a base do sistema. Contém as informações e comportamentos que todo paciente — independente do tipo — deve ter.

Atributos (6 obrigatórios)
  • nome — string · nome completo do paciente
  • data_nascimento — string · data de nascimento no formato "DD/MM/AAAA"
  • cpf — string · CPF do paciente
  • telefone — string · número de contato
  • tipo_sanguineo — string · ex: "A+", "O-", "AB+"
  • numero_prontuario — string · identificador único na clínica
Métodos (mínimo 2)
registrar_atendimento(tipo, custo)
  • Exibe a informação de que o paciente passou por um atendimento do tipo informado, com custo de tanto.
exibir_informacoes(detalhado)
  • Se False → exibe apenas nome, numero_prontuario e tipo_sanguineo
  • Se True → exibe todos os 6 atributos
💡
Boas práticasUse __init__ para receber e inicializar todos os atributos. Prefixe atributos que devem ser privados com _ (ex: self._cpf) para sinalizar encapsulamento.

Subclasse — PacienteParticular

herda de Paciente · pagamento direto pelo atendimento
Subclasse

Herda todos os atributos e métodos de Paciente e adiciona características específicas de pacientes que pagam diretamente pelo atendimento.

ℹ️
Herança obrigatóriaUse super().__init__(...) no __init__ desta classe para inicializar os 6 atributos da superclasse antes de definir os atributos adicionais.
Atributos adicionais (2)
  • forma_pagamento — string · ex: "Cartão", "Pix", "Dinheiro"
  • desconto_fidelidade — float · desconto aplicado por fidelidade (ex: 0.10 = 10%)
Métodos (mínimo 2)
calcular_valor_final(valor_consulta, taxa_urgencia)
  • Adiciona taxa_urgencia ao cálculo caso o atendimento seja de urgência
  • Aplica desconto_fidelidade * valor_consulta sobre o valor
  • Retorna o valor final calculado
exibir_informacoes(detalhado)
  • Sobrescreve o método da superclasse
  • Deve chamar super().exibir_informacoes(detalhado) e acrescentar forma_pagamento e desconto_fidelidade
super().__init__ herança sobrescrita polimorfismo

Subclasse — PacienteConvenio

herda de Paciente · atendimento via plano de saúde
Subclasse

Herda todos os atributos e métodos de Paciente e adiciona características específicas de pacientes atendidos via convênio médico.

ℹ️
Herança obrigatóriaUse super().__init__(...) no __init__ desta classe para inicializar os 6 atributos da superclasse antes de definir os atributos adicionais.
Atributos adicionais (2)
  • nome_convenio — string · nome do plano de saúde (ex: "Unimed", "Amil")
  • numero_carteirinha — string · número da carteirinha do convênio
Métodos (mínimo 2)
registrar_autorizacao(procedimento, valor_glosa)
  • Exibe o procedimento autorizado pelo convênio
  • Exibe o valor da glosa (caso não haja glosa, passar 0)
exibir_informacoes(detalhado)
  • Sobrescreve o método da superclasse
  • Deve chamar super().exibir_informacoes(detalhado) e acrescentar nome_convenio e numero_carteirinha
super().__init__ herança sobrescrita polimorfismo

Dicas para um projeto nota 10
